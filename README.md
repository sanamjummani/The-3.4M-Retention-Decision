## The Problem

Most retention models ask: who is likely to leave? This project asks a different question. That question is worth more.

It is not enough to know who might churn. You also need to know what they are worth, and whether the cost of keeping them is justified by the revenue they generate. A $50,000-per-year property that has a 2% attrition risk is worth intervening on. A $3,000-per-year property needs to be 33% likely to leave before a $1,000 gift makes mathematical sense. Every property has a different bar, and a single universal cutoff gets this wrong.

This project derives that bar individually for each of the 1,312 properties — and produces a clean, actionable output file: gift = 1 or gift = 0, with the math behind every decision.

<!-- METHODOLOGY -->
<table width="100%" cellspacing="0" cellpadding="0" style="border-left:3px solid #C9A84C;background:#0A1628;margin:8px 0">
<tr><td style="padding:10px 16px">
<span style="font-family:monospace;font-size:10px;letter-spacing:3px;color:#C9A84C;text-transform:uppercase">How the Analysis Was Done</span>
</td></tr>
</table>

<br/>

Feature engineering came first — computing annual revenue (ADR × total reservation days), 15% profit, and a composite rating score from the six component ratings. These three derived features carry more signal than the raw variables alone.

Both models were evaluated on the same criterion: 5-fold cross-validated expected dollar profit, with each fold applying the property-specific cutoff rule. This is deliberate. Optimising accuracy or AUC for this problem is wrong — a model that correctly identifies low-value properties as churning but misses high-value ones at risk generates less profit than a model that does the opposite. The evaluation metric matched the business objective from the start.

Random Forest (400 trees in CV, 500 in final training) was selected over Gradient Boosting by $30,942 in cross-validated profit. The final model was trained on all 5,247 training properties and applied to the 1,312 new ones.
